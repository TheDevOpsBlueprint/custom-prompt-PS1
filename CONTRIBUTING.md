# Contributing Guidelines

## PR Rules
- One feature/fix per PR
- Reference an issue in your PR
- Wait for CI checks to pass
- Require at least 1 approval

## Branch Naming
- feat/description
- fix/description
- docs/description
- test/description

## Commit Messages
- feat: new feature
- fix: bug fix
- docs: documentation
- test: adding tests
- refactor: code restructuring
